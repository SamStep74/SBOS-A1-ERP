# v0.7.0 — A1-Validator drift detection + self-hosted CI

## What's new

- **A1-Validator drift detection on vendor bills** (`POST /api/finance/vendor-bills`)
  - Same fail-soft pattern as invoice's customer HVVH re-validation.
  - Re-fetches the vendor's `hvhh` from the `vendors` table at bill-create time (not the
    denormalized value on `purchase_orders.vendor_hvhh`), so a vendor-edit that updated
    the hvhh is caught here.
  - Smoke step 7f: end-to-end proof — create vendor with valid hvhh → bill succeeds →
    directly `UPDATE vendors SET hvhh='NOT_AN_HVVH'` → next bill-create returns 400.

- **A1-Validator on CRM contacts** (`POST /api/finance/crm/contacts`)
  - Self-employed contacts may have their own Armenian TIN distinct from their employer's.
  - New migration `0016_crm_contacts_hvhh.sql` adds the nullable `hvhh TEXT` column to
    `finance.crm_contacts` (plus a partial index).
  - Optional field — most contacts at customer companies don't have their own TIN.
  - Smoke step 7g: valid 8-digit hvhh persists, 9-digit returns 400, omitted is fine.

- **CI self-hosted runner fallback**
  - The org-level hosted-runner quota has been intermittently exhausted since 2026-06-21
    (CI fails with `Job is waiting for a hosted runner to come online` after ~2s with
    empty steps).
  - `runs-on` is now a list: `[self-hosted, sbos-self-hosted, linux, x64, ubuntu-latest]`.
    The first match wins, so a registered self-hosted runner with the `sbos-self-hosted`
    label sidesteps the hosted quota entirely. Falls back to `ubuntu-latest` when no
    self-hosted runner is registered.
  - Full maintainer runbook in `docs/CI.md` (download, configure, systemd).

## New deploy smoke steps

- **STEP 7f** — vendor bill HVVH drift. Creates vendor with valid hvhh → creates catalog item,
  warehouse, location, PO, receives it → creates vendor bill (success) → directly UPDATE
  vendor's hvhh to invalid → restart server → next bill-create returns **400**. End-to-end
  proof of the drift-detection value of the re-validation pass.
- **STEP 7g** — CRM contact TIN. Valid 8-digit hvhh persists, invalid 9-digit returns 400,
  omitted hvhh is fine (optional).

existing rows). The CI workflow change is transparent to PRs — if no self-hosted
runner is registered, behavior is unchanged (still uses hosted `ubuntu-latest`).
