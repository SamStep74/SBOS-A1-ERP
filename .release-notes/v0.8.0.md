# v0.8.0 — A1-Validator on CRM leads + migration renumber

## What's new

- **A1-Validator on CRM leads** (`POST /api/finance/crm/leads`)
  - New migration `0018_crm_leads_hvhh.sql` adds a nullable `hvhh TEXT` column to `finance.crm_leads` (plus a partial index). A lead represents a prospective customer; the hvhh is the prospective company's TIN.
  - Same fail-soft pattern as customer + vendor + invoice + contact: optional field, local regex fallback when A1-Validator is disabled/unreachable.
  - Smoke step 7h: valid 8-digit hvhh persists, invalid 9-digit returns 400, omitted is fine (optional).

- **Migration renumber**
  - `0013_pos_basics.sql` (POS basics from Wave 41) renamed to `0017_pos_basics.sql` to break the duplicate-prefix collision with `0013_catalog_v2.sql`.
  - Both 0013 and 0017 are independent migrations (the POS tables don't FK-reference the catalog tables) so this is a clean rename.
  - The migration_history table records the new filename. Existing deploys that already applied `0013_pos_basics.sql` will see the renamed file as new and re-apply (safe with `IF NOT EXISTS`).

- **Test suite cleanup**
  - `npm install` was never run after the `^5.2.1` Express bump in commit b61ac40 — `node_modules/express` was stuck on 4.22.2. Ran `npm install` to pick up Express 5.2.1.
  - Lint cleanups: `no-useless-assignment` in `server/finance/lots.js` (use `i` then increment), `no-unused-vars` in `server/finance/pos.js` (3 validators prefixed with `_`).
